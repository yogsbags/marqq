<interfaces>
`agent_signals` row lifecycle for this plan:
1. producer inserts `status='pending'`
2. engine claims rows for processing
3. engine loads matching signal trigger config
4. engine reads MKG metrics/baselines and evaluates the configured diff operator
5. engine marks row `triggered`, `ignored`, or `failed`

HooksEngine responsibilities in this plan:
- load config from `hooks-config.js`
- poll Supabase `agent_signals` every `heartbeat_seconds`
- compute diff-from-baseline using MKG values
- generate structured dispatch jobs containing trigger metadata for Plan 05-03
- do not call the agent run endpoint yet; that wiring belongs to Plan 05-03

Recommended exported helper contract:
```javascript
evaluateSignalAgainstBaseline({
  currentValue,
  baselineValue,
  operator,
  threshold,
})
=> { matched, deltaPct, baselineValue, currentValue, reason }
```
</interfaces>
