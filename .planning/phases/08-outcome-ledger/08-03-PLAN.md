## Task 1: Outcomes REST endpoint

1. Add `/api/outcomes/:companyId` to `backend-server.js` that queries `outcome_ledger` for the requested company.
2. Aggregate per-agent accuracy: `accuracy = 1 - min(1, |variance_pct| / 100)` and return `{ agent, accuracy, last_verified, variance_pct }`.
3. Support query params `days=7|30|90` to limit ledger rows; default 30.

## Task 2: Inject calibration notes into run prompts

1. Use `getLatestCalibrationNote(agent, companyId)` when building `systemPrompt` in `/api/agents/:name/run`.
2. Append the note to the `skillsBlock` or as a new section after `memory`.
3. Ensure `test-mode` runs also include the note so tests confirm the note is persisted.

## Task 3: Outcome API + trigger metadata tests

1. Create `platform/content-engine/outcome-api.test.js` to:
   - Seed a fake `outcome_ledger` row and hit the new endpoint.
   - Assert the `/api/outcomes/:companyId` JSON includes the expected accuracy and agent entries.
   - Trigger a signal run and confirm `artifact.data.trigger_context` carries `trigger_metadata`.
2. Cover edges such as missing company_id (400) and `days=90` filtering.
